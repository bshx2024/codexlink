# 亚马逊算法体系概览

## 1. A9/A10 算法（传统搜索逻辑）

### 1.1 核心目标
- **相关性匹配**：将用户搜索词与产品Listing进行匹配。
- **转化率优化**：优先展示转化率高的产品。
- **客户满意度**：考虑评论、退货率等因素。

### 1.2 关键排名因素
1. **关键词匹配度**
   - 标题中的关键词权重最高
   - 五点描述次之
   - 产品描述和后台关键词权重较低
2. **销售表现**
   - 销售速度（Velocity）
   - 转化率（CVR）
   - 销售历史
3. **客户反馈**
   - 评分（Star Rating）
   - 评论数量和质量
   - 退货率
4. **价格竞争力**
   - 价格与竞品对比
   - 购物车占比（Buy Box %）
5. **库存与物流**
   - FBA使用情况
   - 库存深度
   - 配送速度

### 1.3 Listing优化要点
- **标题结构**：品牌 + 核心关键词 + 关键特性 + 差异化
- **关键词布局**：高搜索量词放标题，长尾词放五点，语义词放描述
- **可读性**：避免关键词堆砌，确保人类可读
- **后端搜索词**：补充未在前端使用的同义词、拼写变体、西班牙语关键词等

## 2. Cosmo算法（内容理解与语义匹配）

### 2.1 核心特点
- **语义理解**：不仅匹配关键词，更理解产品含义和用户意图。
- **上下文感知**：考虑用户搜索历史、浏览行为、购买历史。
- **多维度匹配**：结合产品属性、用户画像、场景需求进行匹配。

### 2.2 对Listing的影响
1. **自然语言表达**
   - 使用完整的句子而非关键词堆砌
   - 描述产品如何解决用户问题
   - 包含使用场景和用户利益
2. **语义相关性**
   - 使用同义词和相关概念
   - 描述产品的各种属性和特性
   - 包含用户可能搜索的各种表达方式
3. **内容深度**
   - 提供详细的产品信息
   - 解答用户可能的疑问
   - 包含比较和对比信息

### 2.3 优化策略
- 在描述中使用问答式表达（如"Looking for...?"）
- 包含产品的使用步骤和场景
- 提供详细的规格和参数
- 使用故事化表达增强情感连接

## 3. Rufus AI搜索（新兴AI搜索逻辑）

### 3.1 核心能力
- **对话式搜索**：用户可以使用自然语言提问。
- **意图识别**：理解用户背后的真正需求。
- **个性化推荐**：基于用户特征和上下文提供个性化结果。

### 3.2 对Listing的新要求
1. **问答式内容**
   - 预测用户可能的问题并在Listing中回答
   - 使用"Need..."、"Want..."、"Looking for..."等开头
   - 提供解决方案式的描述
2. **场景化描述**
   - 描述产品在具体场景中的使用
   - 包含用户使用前后的对比
   - 提供多种使用场景示例
3. **差异化价值**
   - 明确说明产品如何优于竞品
   - 提供具体的使用案例和数据
   - 包含客户评价中的亮点

### 3.3 针对Rufus的优化技巧
- 在标题中包含常见问题的关键词
- 在五点描述中使用"解决...问题"、"适用于..."等表达
- 在描述中包含"为什么选择我们"的部分
- 使用具体的数字和数据支持价值主张

## 4. 多算法协同优化策略

### 4.1 内容层次设计
- **第一层（A9优先）**：核心关键词密集布局在标题和前两点
- **第二层（Cosmo适配）**：自然语言描述，同义词和相关概念分布在五点和描述
- **第三层（Rufus优化）**：问答式表达、场景描述、差异化价值

### 4.2 关键词矩阵
| 算法 | 关键词类型 | 放置位置 | 密度建议 |
|------|-----------|---------|---------|
| A9/A10 | 核心关键词 | 标题、前两点 | 高（但自然） |
| Cosmo | 语义相关词 | 五点、描述 | 中 |
| Rufus | 问题式关键词 | 标题、描述 | 中低 |

### 4.3 测试与迭代
1. **A/B测试**：定期测试不同版本的Listing
2. **数据分析**：监控关键词排名、转化率、销售变化
3. **竞品对比**：分析竞品在哪些算法上表现更好
4. **持续优化**：根据数据反馈调整关键词布局和内容表达
